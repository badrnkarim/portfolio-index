# Contributing

Thanks for your interest.

## Scope
This is a portfolio project. Contributions are welcome if they improve:
- Clarity and structure
- Formatting consistency
- Workflow correctness (assurance style)

## How to contribute
1) Open an Issue describing the improvement
2) Submit a PR explaining:
   - What changed
   - Why it changed

## Rules
- Keep changes small and reviewable
- Do not add secrets or sensitive data
- Use clear filenames and consistent headings

## Security
Do not disclose vulnerabilities publicly. See SECURITY.md.
